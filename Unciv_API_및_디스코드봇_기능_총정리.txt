작성일: 2026-04-17

==================================================
0) 문서 범위와 기준
==================================================
- 이 문서는 Unciv 멀티플레이 네트워크 API를 중심으로 정리했습니다.
- 기준 소스:
  1) Unciv 공식 문서의 Multiplayer 페이지
  2) yairm210/Unciv 공개 소스의 multiplayer 관련 코드
  3) UncivServer(기본 서버) 라우팅 코드
- 검증 상태:
  - 2026-04-17 기준 main 브랜치 코드와 대조하여 엔드포인트/이벤트/오류코드 항목을 재검증했습니다.
- 주의:
  - Unciv는 단일 "공식 API 포털"이 따로 있는 형태가 아니라, 서버 구현 버전(APIv0/v1/v2)에 따라 인터페이스가 다릅니다.
  - 따라서 "모든 API"는 실질적으로 아래 3종 인터페이스를 모두 포함하는 의미로 이해하는 것이 정확합니다.


==================================================
1) Unciv API 버전 체계 (핵심)
==================================================
1. APIv0 (Dropbox 백엔드)
- 성격: 파일 저장소 기반(게임 파일 업/다운로드 중심)
- 특징: 고급 기능(로비/친구/초대/REST 리소스 모델/WS 이벤트)이 없음
- Unciv 내부에서 사용되는 Dropbox 엔드포인트:
  - https://content.dropboxapi.com/2/files/upload
  - https://content.dropboxapi.com/2/files/download
  - https://api.dropboxapi.com/2/files/get_metadata
  - https://api.dropboxapi.com/2/files/delete_v2
  - https://api.dropboxapi.com/2/file_properties/templates/add_for_user

2. APIv1 (UncivServer 기본 서버, 단순 HTTP + 채팅 WS)
- 성격: 파일 기반 멀티플레이 서버
- 특징:
  - /files/{fileName} 중심 업로드/다운로드
  - /auth로 인증 상태 조회/비밀번호 설정
  - /chat WebSocket으로 채팅/구독
  - /isalive로 서버 기능(authVersion/chatVersion) 제공

3. APIv2 (확장 REST + WebSocket)
- 성격: 리소스 중심 REST API + 실시간 이벤트 WS
- 특징:
  - account/auth/friend/game/invite/lobby/chat 전체 지원
  - /api/version으로 버전 확인
  - /api/v2/ws로 이벤트 수신
  - game_data_id 기반 증분 동기화


==================================================
2) API 자동 감지 규칙 (Unciv 클라이언트 기준)
==================================================
- 1차: /isalive 확인 성공 시 APIv1로 판정
- 2차: 실패 시 /api/version 확인 성공 + version 응답 파싱 시 APIv2 판정
- 별도: Dropbox 기본 URL이면 APIv0 처리


==================================================
3) APIv1 상세 (UncivServer 기본 서버)
==================================================

3-1. HTTP 엔드포인트
A) GET /isalive
- 목적: 서버 생존 확인 + 기능 버전 확인
- 대표 응답:
  - JSON 예시: {"authVersion": 1, "chatVersion": 1}
  - 일부 구현은 단순 true 텍스트만 반환할 수도 있음

B) PUT /files/{fileName}
- 목적: 파일 업로드
- 인증: Basic Auth 사용(서버 설정에 따라 필수)
- 용도:
  - gameId 파일 저장(전체 게임 상태)
  - gameId_Preview 파일 저장(프리뷰 상태)

C) GET /files/{fileName}
- 목적: 파일 다운로드
- 실패: 파일 없음이면 404

D) GET /auth
- 목적: 인증 상태 조회
- 노출 조건: authV1Enabled=true일 때만 라우트 생성
- 대표 상태코드:
  - 200: 인증됨(비밀번호 일치)
  - 204: 계정 등록 전/비밀번호 미설정 상태
  - 401: 인증 실패

E) PUT /auth
- 목적: 비밀번호 설정/변경
- 노출 조건: authV1Enabled=true일 때만 라우트 생성
- 규칙:
  - 기존 비밀번호가 있으면 old 비밀번호 확인 후 변경
  - 길이 제한(최소 6자) 존재

3-2. WebSocket 엔드포인트
F) WS /chat
- 목적: 채팅 및 게임 채널 구독
- 노출 조건: chatV1Enabled=true일 때만 라우트 생성
- 인증 규칙: Basic 인증 정보가 없거나 서버 비밀번호와 불일치하면 error 메시지 후 연결 종료
- 클라이언트 -> 서버 메시지 타입:
  1) chat: { civName, message, gameId }
  2) join: { gameIds: ["..."] }
  3) leave: { gameIds: ["..."] }
- 서버 -> 클라이언트 메시지 타입:
  1) chat: { civName, message, gameId? }
  2) joinSuccess: { gameIds: ["..."] }
  3) error: { message }

3-3. APIv1 서버 운영 옵션
- 기본 포트: 8080
- 실행 옵션 예시:
  - -p / -port (포트)
  - -f / -folder (저장 폴더)
  - -a / -auth 또는 -no-auth (인증 켜기/끄기)
  - -c / -chat 또는 -no-chat (채팅 켜기/끄기)

4) APIv2 상세 (REST + WS)
==================================================

4-1. 버전/호환성 엔드포인트
A) GET /api/version
- 목적: 서버 API 버전 확인
- 응답 모델: VersionResponse { version: Int }
- 예: version = 2

B) WS /api/v2/ws
- 목적: 실시간 이벤트 스트림
- 인증: 세션 쿠키 필요

4-2. Accounts API
A) GET /api/v2/accounts/me
- 목적: 내 계정 정보 조회

B) PUT /api/v2/accounts/me
- 목적: 계정 정보 수정(username/display_name)

C) DELETE /api/v2/accounts/me
- 목적: 계정 삭제(복구 불가)

D) GET /api/v2/accounts/{uuid}
- 목적: UUID 기반 계정 조회

E) POST /api/v2/accounts/lookup
- 목적: username -> 계정 조회
- 요청: LookupAccountUsernameRequest

F) POST /api/v2/accounts/register
- 목적: 계정 생성
- 요청: AccountRegistrationRequest

G) POST /api/v2/accounts/setPassword
- 목적: 비밀번호 변경
- 요청: SetPasswordRequest { old_password, new_password }

4-3. Auth API
A) POST /api/v2/auth/login
- 목적: 로그인 및 세션 발급
- 요청: LoginRequest
- 세션 쿠키 이름: id

B) GET /api/v2/auth/logout
- 목적: 로그아웃 및 세션 무효화

4-4. Chat API
A) GET /api/v2/chats
- 목적: 접근 가능한 채팅방 목록 조회
- 응답: friend_chat_rooms, game_chat_rooms, lobby_chat_rooms

B) GET /api/v2/chats/{roomUUID}
- 목적: 특정 채팅방 메시지 조회

C) POST /api/v2/chats/{chatRoomUUID}
- 목적: 채팅 메시지 전송
- 요청: SendMessageRequest { message }

4-5. Friend API
A) GET /api/v2/friends
- 목적: 친구 목록 + 친구요청 목록 조회

B) POST /api/v2/friends
- 목적: 친구 요청 생성
- 요청: CreateFriendRequest { uuid }

C) PUT /api/v2/friends/{friendRequestUUID}
- 목적: 친구 요청 수락

D) DELETE /api/v2/friends/{friendUUID}
- 목적: 친구/요청 삭제(상태에 따라 정리)

4-6. Game API
A) GET /api/v2/games
- 목적: 참여 중인 게임 개요 목록

B) GET /api/v2/games/{gameUUID}
- 목적: 단일 게임 상태 조회(full game_data 포함)

C) PUT /api/v2/games/{gameUUID}
- 목적: 게임 상태 업로드
- 요청: GameUploadRequest { game_data }
- 응답: GameUploadResponse { game_data_id }

4-7. Invite API
A) GET /api/v2/invites
- 목적: 초대 목록 조회

B) POST /api/v2/invites
- 목적: 로비 초대 생성
- 요청: CreateInviteRequest { friend_uuid, lobby_uuid }

C) DELETE /api/v2/invites/{inviteUUID}
- 목적: 초대 거절/철회

4-8. Lobby API
A) GET /api/v2/lobbies
- 목적: 로비 목록 조회

B) POST /api/v2/lobbies
- 목적: 로비 생성
- 요청: CreateLobbyRequest { name, password?, max_players }

C) GET /api/v2/lobbies/{lobbyUUID}
- 목적: 단일 로비 상세 조회

D) DELETE /api/v2/lobbies/{lobbyUUID}
- 목적: 로비 종료(오너)

E) DELETE /api/v2/lobbies/{lobbyUUID}/{playerUUID}
- 목적: 로비에서 특정 플레이어 강퇴

F) POST /api/v2/lobbies/{lobbyUUID}/join
- 목적: 로비 참가
- 요청: JoinLobbyRequest { password? }

G) POST /api/v2/lobbies/{lobbyUUID}/leave
- 목적: 로비 나가기

H) POST /api/v2/lobbies/{lobbyUUID}/start
- 목적: 로비를 게임으로 시작
- 응답: StartGameResponse { game_chat_uuid, game_uuid }

5) APIv2 핵심 데이터 모델 요약
==================================================
- AccountResponse: username, display_name, uuid
- GameOverviewResponse:
  - game_uuid
  - game_data_id
  - chat_room_uuid
  - last_activity
  - last_player
  - max_players
  - name
- GameStateResponse:
  - game_data(문자열)
  - game_data_id
  - 메타 필드(채팅방/플레이어/시간 등)
- ChatMessage:
  - uuid
  - sender
  - message
  - created_at
- Invite / Lobby / Friend 관련 모델은 UUID + 생성시간 + 사용자 정보를 중심으로 구성

==================================================
8) 게임 데이터 인코딩/파싱 규칙
==================================================
- Unciv 저장 문자열은 기본적으로 다음 구조를 사용:
  1) JSON 직렬화
  2) GZIP 압축
  3) Base64 인코딩
- 역직렬화 시에는 gzip 해제 시도 후 실패하면 평문 JSON으로 처리하는 호환 경로를 가짐
- 멀티플레이 파일명 관례(APIv1 계열):
  - 본문: gameId
  - 프리뷰: gameId_Preview


==================================================
9) 디스코드 봇에 넣을 수 있는 기능 총정리 (상세)
==================================================
아래 목록은 "실제로 운영 가능한 기능" 기준으로 최대한 포괄해 정리했습니다.
각 기능은 구현 시 필요한 API 버전 조건을 함께 표기했습니다.

[표기]
- 필요 API: v1 / v2 / 공통
- 난이도: 하 / 중 / 상

A. 기본 운영 기능
1) 봇 상태 점검 (/ping)
- 설명: 게이트웨이 지연과 봇 살아있음 확인
- 필요 API: 공통
- 난이도: 하

2) 서버 연결 진단 (/unciv health)
- 설명: 대상 멀티플레이 서버의 /isalive 또는 /api/version 응답 확인
- 필요 API: v1 또는 v2
- 난이도: 하

3) API 버전 자동 판별 (/unciv api-detect)
- 설명: v0/v1/v2를 자동 판별해 이후 기능 분기
- 필요 API: 공통
- 난이도: 중

B. 게임 조회/상태 기능
4) 게임 개요 조회 (/unciv game status <id>)
- 설명: 턴, 현재 플레이어, 최근 활동 시각, 점수표 출력
- 필요 API: v1(파일 파싱) 또는 v2(/games)
- 난이도: 중

5) 현재 턴 플레이어 표시 (/unciv game current <id>)
- 설명: 누가 턴 진행 중인지 빠르게 확인
- 필요 API: v1/v2
- 난이도: 중

6) 리더보드 출력 (/unciv game leaderboard <id>)
- 설명: 문명별 점수/순위 표시
- 필요 API: v1/v2(게임 데이터 파싱)
- 난이도: 중

7) 특정 문명 상세 보기 (/unciv game civ <id> <civ>)
- 설명: 점수, 과학, 군사력 등 문명 중심 요약
- 필요 API: v1/v2
- 난이도: 상

8) 게임 비교 (/unciv game diff <id>)
- 설명: 마지막 저장 상태 대비 변경사항(턴, 점수, 주도권)
- 필요 API: v1/v2
- 난이도: 상

C. 멀티 게임 관리 기능
9) 게임 등록 (/unciv track add <game_id> [별칭])
- 설명: 서버별 추적 게임 목록에 등록
- 필요 API: 공통
- 난이도: 하

10) 게임 해제 (/unciv track remove <game_id>)
- 설명: 추적 목록 제거
- 필요 API: 공통
- 난이도: 하

11) 추적 목록 (/unciv track list)
- 설명: 서버/채널별 추적 게임 전체 표시
- 필요 API: 공통
- 난이도: 하

12) 별칭 변경 (/unciv track alias <game_id> <name>)
- 설명: 사람이 읽기 좋은 게임 이름 관리
- 필요 API: 공통
- 난이도: 하

D. 자동 알림 기능
13) 턴 변경 알림
- 설명: 현재 플레이어가 바뀌면 채널에 자동 공지
- 필요 API: v1/v2
- 난이도: 중

14) 내 차례 알림(역할 멘션)
- 설명: 플레이어-디스코드 사용자 매핑을 통해 개인 알림
- 필요 API: v1/v2
- 난이도: 중

15) 장기 미진행 게임 경고
- 설명: N시간 이상 변화 없는 게임 경고
- 필요 API: v1/v2
- 난이도: 중

16) 서버 장애 알림
- 설명: 일정 횟수 연속 실패 시 장애 공지 + 자동 복구 알림
- 필요 API: 공통
- 난이도: 중

17) 스팸 방지 중복 억제
- 설명: 동일 상태 반복 알림 방지(해시 기반)
- 필요 API: 공통
- 난이도: 중

E. APIv2 전용 협업 기능
18) 계정 로그인/로그아웃 (/uncivv2 auth login|logout)
- 설명: 세션 쿠키 기반 인증 관리
- 필요 API: v2
- 난이도: 중

19) 내 계정 조회 (/uncivv2 account me)
- 설명: username/display_name/uuid 조회
- 필요 API: v2
- 난이도: 하

20) 친구 관리 (/uncivv2 friend list|add|accept|remove)
- 설명: 친구요청 생성/수락/삭제 자동화
- 필요 API: v2
- 난이도: 중

21) 로비 목록 조회 (/uncivv2 lobby list)
- 설명: 공개 로비/정원/비밀번호 여부 조회
- 필요 API: v2
- 난이도: 하

22) 로비 생성/시작 (/uncivv2 lobby open|start)
- 설명: 로비 생성 후 게임 시작까지 워크플로우 제공
- 필요 API: v2
- 난이도: 중

23) 로비 참여/퇴장/강퇴 (/uncivv2 lobby join|leave|kick)
- 설명: 운영 편의 기능
- 필요 API: v2

12) 결론
==================================================
- Unciv의 멀티플레이 API는 APIv0/v1/v2 3축으로 이해하는 것이 정확합니다.
- 디스코드 봇은 단순 조회형에서 시작해, APIv2 + WS를 붙이면 협업/실시간 운영 도구로 확장할 수 있습니다.
- 위 50개 기능 목록을 기반으로 단계별 구현하면 실제 운영 가능한 Unciv 커뮤니티 봇을 만들 수 있습니다.

13) 사실 검증 근거 (원문 소스 매핑)
==================================================
[A] API 버전 감지
- 근거 파일: core/src/com/unciv/logic/multiplayer/ApiVersion.kt
- 핵심 확인:
  - Dropbox URL 일치 시 APIv0
  - /isalive 성공(또는 true 텍스트) 시 APIv1
  - /api/version 성공 + VersionResponse 파싱 시 APIv2

[B] APIv1 서버 라우트
- 근거 파일: server/src/com/unciv/app/server/UncivServer.kt
- 핵심 확인:
  - GET /isalive
  - GET/PUT /files/{fileName}
  - GET/PUT /auth (authV1Enabled일 때)
  - WS /chat (chatV1Enabled일 때)
  - /auth 상태코드 200/204/401 분기

[C] APIv2 REST 엔드포인트
- 근거 파일: core/src/com/unciv/logic/multiplayer/apiv2/EndpointImplementations.kt
- 핵심 확인:
  - accounts/auth/chats/friends/games/invites/lobbies 전 경로
  - 예: api/v2/auth/login, api/v2/games/{gameUUID}, api/v2/lobbies/{lobbyUUID}/start

[D] APIv2 오류 코드
- 근거 파일: core/src/com/unciv/logic/multiplayer/apiv2/ResponseStructs.kt
- 핵심 확인:
  - ApiStatusCode 1000~1023, 2000~2002

[E] APIv2 WebSocket 이벤트
- 근거 파일: core/src/com/unciv/logic/multiplayer/apiv2/WebSocketStructs.kt
- 핵심 확인:
  - invalidMessage, gameStarted, updateGameData, ... , accountUpdated
  - updateGameData.game_data 주석: base64-encoded, gzipped game state

[F] 저장 데이터 인코딩 규칙
- 근거 파일:
  - core/src/com/unciv/ui/screens/savescreens/Gzip.kt
  - core/src/com/unciv/logic/files/UncivFiles.kt
- 핵심 확인:
  - gameInfoToString: JSON -> (옵션/강제) GZIP+Base64
  - gameInfoFromString: Gzip.unzip 시도 후 실패 시 평문 JSON 처리

[G] APIv0 Dropbox 엔드포인트
- 근거 파일: core/src/com/unciv/logic/multiplayer/storage/DropBox.kt
- 핵심 확인:
  - files/upload, files/download, files/get_metadata, files/delete_v2
  - file_properties/templates/add_for_user (코드상 주석 처리되어 있으나 URL 근거 존재)

14) 현재 봇 구현 반영 상태 (2026-04-17)
==================================================
- 구현 완료 (1단계 핵심)
  - 서버 헬스체크: !unciv health, /unciv health
  - 게임 추적 등록/해제/목록: !unciv track add|remove|list, /unciv track add|remove|list
  - 턴 변경 자동 알림: 추적된 게임을 주기 폴링하여 턴/현재 플레이어 변동 시 채널에 자동 임베드 전송

- 유지된 기존 기능
  - 상태 조회: !unciv game, !unciv status, /unciv game, /unciv status
  - 기본 응답: !ping, /ping

- 새 환경변수(선택)
  - UNCIV_TRACK_POLL_INTERVAL_SEC: 추적 폴링 주기(기본 90초, 최소 30초, 최대 1800초)
  - UNCIV_TRACK_FILE: 추적 목록 저장 파일 경로(기본 tracked_games.json)
